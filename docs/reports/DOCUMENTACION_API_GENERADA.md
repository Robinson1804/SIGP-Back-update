## Mantenimiento Futuro

### Cuando Agregues un Nuevo Endpoint

1. Agregar a `openapi.yaml`
2. Documentar en `API_REFERENCE.md`
3. Agregar ejemplo en `EXAMPLES.md`
4. Actualizar si es necesario:
   - `README.md`
   - `AUTHENTICATION.md` (si afecta permisos)
   - `DOCUMENTATION_SUMMARY.md`

### Cuando Hagas Breaking Changes

1. Crear nueva versión en `openapi.yaml`
   ```yaml
   servers:
     - url: /api/v2
   ```
2. Agregar CHANGELOG
3. Crear documentación v2.0.0
4. Mantener v1 en rama compatibilidad
